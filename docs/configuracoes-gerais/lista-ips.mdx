Para garantir o correto funcionamento das integrações com a Robbu, alguns **endereços IP** precisam estar **liberados no firewall** ou servidor do cliente.

**Por que liberar IPs:**

- 🔗 **Integrações:** Comunicação com APIs externas
- 📡 **Webhooks:** Envio de eventos em tempo real
- 📁 **FTP/SFTP:** Leitura e envio de arquivos
- 🔧 **Serviços customizados:** Integrações dedicadas

Esta documentação organiza os IPs de forma simplificada, agrupando IPs compartilhados e destacando os serviços que os utilizam.

## Como Liberar IPs no Firewall

<Steps>
  <Step title="Identificar serviços usados">
    Determine quais serviços da Robbu seu ambiente utiliza:
    
    - ✅ Invenio IDR (V1 ou V2)?
    - ✅ Webhooks (OutputStream)?
    - ✅ Jobs (importação/exportação)?
    - ✅ Integração FTP/SFTP?
    - ✅ Integrações customizadas?
  </Step>
  
  <Step title="Copiar lista de IPs correspondente">
    Com base nos serviços identificados, copie as listas de IPs das abas acima.
    
    **Dica:** Se houver dúvida, libere **todos os IPs** documentados para garantir compatibilidade total.
  </Step>
  
  <Step title="Configurar firewall">
    Configure seu firewall, WAF ou lista de ACL para **permitir tráfego de entrada** dos IPs da Robbu.
    
    **Configurações recomendadas:**
    
    - **Protocolo:** TCP
    - **Portas:** Conforme serviço (ex: 443 para HTTPS, 21/22 para FTP/SFTP)
    - **Direção:** Inbound (entrada)
    - **Ação:** Permitir (Allow)
    
    <Warning>
    **Importante:** Configure as regras como **"Source Address"** ou **"Source IP"**, não como destino.
    </Warning>
  </Step>
  
  <Step title="Testar conectividade">
    Após liberar os IPs:
    
    1. Teste a integração específica (webhook, FTP, API)
    2. Verifique logs do firewall se houver falhas
    3. Confirme que não há bloqueios residuais
    
    <Check>
    **Validação:** Entre em contato com o suporte Robbu para confirmar que a conectividade está funcionando corretamente.
    </Check>
  </Step>
</Steps>

## Observações Importantes

<Note>
**Considerações gerais:**

- 📋 Para clientes que utilizam **múltiplos serviços**, basta liberar a **lista combinada dos IPs** de cada grupo
- ♻️ Serviços que **compartilham IPs** (IDR V1, OutputStream, Jobs) requerem liberação **única**
- ✅ Em caso de dúvida, recomenda-se **liberar todos os IPs** listados para garantir total compatibilidade
- 🔄 Esta lista **pode ser atualizada** conforme evolução da infraestrutura Robbu
- 📅 **Revise periodicamente** esta documentação para mudanças
</Note>

<Warning>
**Segurança:**

Liberar IPs no firewall é seguro **quando feito corretamente**:

- ✅ Libere **apenas** nas portas necessárias (não todas)
- ✅ Documente as regras criadas
- ✅ Revise periodicamente a lista
- ✅ Use em conjunto com autenticação forte (tokens, certificados)
</Warning>

## ⁉️ Perguntas Frequentes (FAQ)

<AccordionGroup>
  <Accordion title="🔐 O que é a liberação de IPs da Robbu?">
    **Liberação de IPs** é o processo de **permitir** que os servidores da Robbu se comuniquem com o ambiente do cliente.
    
    **Envolve:**
    
    - 🛡️ **Firewall:** Adicionar IPs à lista de permissões (allowlist/whitelist)
    - 🌐 **WAF:** Configurar Web Application Firewall para aceitar tráfego
    - 🔧 **ACL:** Ajustar Access Control Lists em servidores
    
    **Por que é necessário:**
    
    Muitos ambientes corporativos possuem **firewalls restritivos** que bloqueiam conexões de IPs desconhecidos por padrão. Para que serviços da Robbu funcionem, é preciso **explicitamente permitir** nossos IPs.
    
    **Analogia:** É como adicionar um contato à lista VIP de um condomínio - sem isso, a portaria (firewall) não deixa entrar.
  </Accordion>

  <Accordion title="⏱️ Quando é necessário realizar a liberação de IPs?">
    **A liberação é necessária sempre que:**
    
    **1. Integrações via API** 🔗
    - Robbu precisa chamar APIs no seu ambiente
    - Seu servidor API tem firewall
    
    **2. Webhooks (OutputStream)** 📡
    - Robbu envia eventos para um endpoint seu
    - Seu servidor que recebe webhooks tem firewall
    
    **3. Integrações FTP/SFTP** 📁
    - Robbu acessa seu servidor FTP para ler/escrever arquivos
    - Seu servidor FTP tem restrições de IP
    
    **4. Firewalls restritivos** 🛡️
    - Política de "deny by default" (bloqueia tudo por padrão)
    - Apenas IPs autorizados são permitidos
    
    **5. Comunicação falha** ❌
    - Serviços da Robbu não conseguem alcançar sistemas internos
    - Logs mostram "connection refused" ou "timeout"
    
    **Como saber se preciso liberar:**
    - Teste a integração → Se falhar com erro de rede, provavelmente é firewall
    - Consulte sua equipe de TI/infra sobre políticas de firewall
  </Accordion>

  <Accordion title="📋 Quais serviços da Robbu exigem liberação de IPs?">
    **Principais serviços que requerem liberação:**
    
    **1. Invenio IDR (V1 e V2)** 🔧
    - **Função:** Motor de regras que processa e roteia mensagens
    - **Precisa acessar:** APIs externas, CRMs, ERPs
    - **IPs:** 1 IP (V2) ou 20 IPs (V1)
    
    **2. OutputStream (Webhooks)** 📡
    - **Função:** Envia eventos em tempo real para sistemas externos
    - **Precisa acessar:** Seu endpoint de webhook
    - **IPs:** 20 IPs (mesmos do IDR V1)
    
    **3. Invenio Jobs (V1 e V2)** 📅
    - **Função:** Processa jobs agendados (importação/exportação)
    - **Precisa acessar:** Servidor FTP/SFTP do cliente
    - **IPs:** 20 IPs (mesmos do IDR V1)
    
    **4. Invenio Center** 🌐
    - **Função:** Interface administrativa, teste de conexões FTP
    - **Precisa acessar:** Servidor FTP para validar credenciais
    - **IPs:** 1 IP dedicado (`104.41.0.210`)
    
    **5. InvenioCustom** 🔧
    - **Função:** Integrações dedicadas (SOAP, WCF, orquestrações)
    - **Precisa acessar:** APIs/serviços específicos do cliente
    - **IPs:** 24 IPs exclusivos
    
    **Resumo:**
    - Se usa **qualquer integração** com sistemas externos → Precisa liberar IPs
    - Se usa **apenas** Invenio Live/Center sem integrações → Pode não precisar

    **Por quê fazer isso:**
    
    - ✅ **Garantia total:** 100% de compatibilidade com todos os serviços
    - ✅ **Prova de futuro:** Se novos serviços forem ativados, já estarão funcionando
    - ✅ **Menos troubleshooting:** Evita diagnósticos demorados
    - ✅ **Simplicidade:** Não precisa entender cada serviço em detalhes
    
    **Total de IPs únicos:**
    - **Invenio IDR V2:** 1 IP
    - **Invenio IDR V1/OutputStream/Jobs:** 20 IPs
    - **FTP test:** 1 IP
    - **InvenioCustom:** 24 IPs

    **Como fazer:**
    
    1. Copie **todas as listas** deste documento
    2. Cole no seu sistema de firewall (remova duplicatas se necessário)
    3. Configure para **permitir** (allow/accept) nas portas relevantes
    
    **Portas comuns:**
    - 443 (HTTPS) - Webhooks, APIs
    - 21/22 (FTP/SFTP) - Transferência de arquivos
    - 80 (HTTP) - Se APIs não usam HTTPS
    
    <Warning>

    - Use em conjunto com **autenticação** (tokens, chaves)
    - Limite a **portas específicas** (não libere TODAS as portas)
    - **Documente** o que foi liberado e por quê
