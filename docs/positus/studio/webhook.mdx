# Webhook

Para configurar o webhook do workspace acesse: [https://studio.posit.us/workspace/configuracoes](https://studio.posit.us/workspace/configuracoes)

Todo evento entregue ao seu webhook segue o mesmo envelope: um campo `event` com o nome do evento, um objeto `workspace` identificando o workspace de origem e um objeto com o recurso relacionado ao evento (`number`, `template` etc.). A entrega é feita via HTTP `POST`.

**Objeto `workspace`**

| Campo            | Descrição                                                     |
| ---------------- | ------------------------------------------------------------- |
| id               | Identificador (UUID) do workspace                             |
| provider         | Provedor do workspace: `{ "id", "name" }`                     |
| type_workspace   | Tipo do workspace: `{ "id", "name" }` (Receptivo / Prospect) |
| name             | Nome do workspace                                             |
| business_id      | ID do Business Manager na Meta                                |
| waba_id          | ID da WABA na Meta                                            |

## Classificação de qualidade do número

Receba em seu webhook a qualidade de cada número de WhatsApp afim de manter uma melhor integração com o seu CRM, aumentando assim a qualidade da gestão em seus números.

* Indefinido: ainda não qualificado
* Verde: qualidade alta
* Amarelo: qualidade média
* Vermelho: qualidade baixa

### Eventos de número

| Tipo de Evento              | Descrição                                                                                                                                             |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| phone_number_status_update  | Enviado em tempo real sempre que o status, a classificação de qualidade (`quality_rating`) ou o limite de mensagens (`messaging_limit`) do número é alterado. |
| phone_number_quality_update | Enviado durante a sincronização periódica dos números. Utiliza um payload em formato achatado (ver exemplo abaixo).                                    |
| phone_number_sync           | Enviado durante a sincronização periódica, refletindo o estado atual do número.                                                                       |

### Status do número

| **Status do número** | Descrição                                                     |
| -------------------- | ------------------------------------------------------------- |
| 1                    | Número aguardando aprovação                                   |
| 2                    | Número aguardando criação do servidor                        |
| 3                    | Número aguardando autenticação do servidor                   |
| 4                    | Número aguardando instalação do servidor                     |
| 5                    | Número aguardando configuração do servidor                   |
| 6                    | Número aguardando ativação                                    |
| 7                    | Número ativo                                                  |
| 8                    | Número inativo                                                |
| 9                    | Número excluído                                               |
| 10                   | Aguardando migração para Cloud Api                            |
| 11                   | Número aguardando reinicialização do servidor                |
| 12                   | Número aguardando finalização da ativação por parte do usuário |
| 13                   | Número banido                                                 |
| 14                   | Número aguardando restauração                                 |
| 15                   | Número transferido                                            |

### Status de qualidade (`quality_status`)

| ID | Código    |
| -- | --------- |
| 0  | UNKNOWN   |
| 1  | CONNECTED |
| 2  | FLAGGED   |
| 3  | MIGRATED  |

### Classificação de qualidade (`quality_rating`)

| ID | Descrição da classificação de qualidade |
| -- | --------------------------------------- |
| 0  | UNKNOWN                                 |
| 1  | GREEN                                   |
| 2  | YELLOW                                  |
| 3  | RED                                     |

## Mensagem de template

Receba notificações de aprovação das suas templates de notificações por webhook. O evento `message_template_status_update` é enviado sempre que o status de aprovação de uma template é alterado.

### Status da template

| ID | Código (code)    | Descrição         |
| -- | ---------------- | ----------------- |
| 0  | UNDEFINED        | Indefinido        |
| 1  | PENDING          | Pendente          |
| 2  | APPROVED         | Aprovado          |
| 3  | REJECTED         | Rejeitado         |
| 4  | IN_APPEAL        | Em recurso        |
| 5  | PENDING_DELETION | Exclusão pendente |
| 6  | DELETED          | Excluído          |
| 7  | DISABLED         | Desabilitado      |
| 8  | FLAGGED          | Sinalizado        |
| 9  | REINSTATED       | Reinstalado       |
| 10 | PAUSED           | Pausado           |
| 11 | PAUSED           | Pausado           |
