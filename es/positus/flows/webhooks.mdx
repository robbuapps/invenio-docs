# Webhooks de Flujos

Para hacer seguimiento de las actualizaciones de los flujos, se recomienda configurar un webhook en el workspace para recibir las actualizaciones sobre los flujos en tiempo real. Es necesario [configurar un webhook en el workspace](https://studio.posit.us/workspace/configuracoes) para recibir estos eventos.

## Eventos de Flujos

La tabla siguiente lista todos los eventos de flujos que pueden ser recibidos a través de los webhooks del workspace:

| Evento | Descripción |
| --- | --- |
| flow_created | Este evento se recibe cuando se crea un nuevo flujo |
| flow_updated | Este evento se recibe cuando se actualiza un flujo |
| flow_published | Este evento se recibe cuando se publica un flujo |
| flow_throttled | Este evento se recibe cuando un flujo es restringido |
| flow_deprecated | Este evento se recibe cuando un flujo es descontinuado |
| flow_blocked | Este evento se recibe cuando un flujo es bloqueado |
| flow_deleted | Este evento se recibe cuando un flujo es eliminado |
| flow_sync | Este evento se recibe cuando el sistema de Positus fuerza una sincronización con Meta |

<Info>
Es importante que tu Webhook devuelva una respuesta HTTPS 200 OK a las notificaciones. De lo contrario, el cliente de la API de WhatsApp Business considerará esa notificación como un fallo e intentará nuevamente después de un retraso.
</Info>
