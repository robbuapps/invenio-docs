# Flujos dinámicos

## Endpoint

Cualquier `Flow` que sea de tipo dinámico necesita tener un `endpoint` configurado durante su creación para que pueda realizar intercambios de datos entre las pantallas. Además del intercambio de datos, el `endpoint` también es responsable de gestionar cuál será la siguiente pantalla del flujo.

<Warning>
El `endpoint` debe estar siempre activo y respondiendo. Si el `endpoint` presenta problemas, el `Flow` puede sufrir restricciones por parte de WhatsApp. Obtén más información en la sección sobre [Salud del endpoint / Throttle](#salud-del-endpoint--throttle).
</Warning>

## Webhook

El webhook del número que envió el `Flow` será llamado solo después de que el `Flow` sea completado y enviado por el usuario en una pantalla que tiene la propiedad `terminal: true`.

Ejemplo de webhook a ser completado después del envío por el usuario:

## Salud del endpoint / Throttle

Los flujos con endpoint inestable pueden ser restringidos (throttle) por WhatsApp. Cuando el `endpoint` presenta fallos o respuestas lentas, WhatsApp puede limitar el número de solicitudes permitidas, afectando la disponibilidad del Flow para los usuarios. Monitorea regularmente la salud de tu endpoint para evitar estos bloqueos.
