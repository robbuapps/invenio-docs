## Flujos dinámicos

### Inscripción en curso en línea

Este es un ejemplo de Flow dinámico que implementa un formulario de inscripción en una clase en línea. El usuario completa sus datos personales, confirma la inscripción y, opcionalmente, puede informar cómo conoció el curso.

#### Código del Flow (Flow JSON)

```json
{
	"version":"3.1",
	"data_api_version":"3.0",
	"routing_model":{
	  "REGISTER":[
		"REGISTRATION_SUCCESS",
		"REGISTRATION_DISABLED"
	  ],
	  "REGISTRATION_SUCCESS":[
		
	  ],
	  "REGISTRATION_DISABLED":[
		
	  ]
	},
	"screens":[
	  {
		"id":"REGISTER",
		"title":"Inscribirse en la clase",
		"data":{
		  
		},
		"layout":{
		  "type":"SingleColumnLayout",
		  "children":[
			{
			  "type":"TextHeading",
			  "text":"Inscríbete en nuestra próxima clase"
			},

			},
			{
			  "type":"Form",
			  "name":"form",
			  "children":[
				{
				  "type":"TextInput",
				  "name":"firstName",
				  "label":"Primer nombre",

				  "required":true

				  "type":"Footer",

				  "on-click-action":{
					"name":"data_exchange",

			{

			  "name":"form",

				{
				  "type":"RadioButtonsGroup",

					{
					  "id":"0",

					},

					"payload":{
					  "source":"${form.source}",
					  "firstName":"${data.firstName}",

		"id":"REGISTRATION_DISABLED",
```

#### Comportamiento esperado del endpoint

Para completar este Flow, debes implementar un endpoint en tu servidor que procese los datos del formulario. El comportamiento esperado se describe en la página de [Flujos dinámicos](/es/positus/flows/flujos-dinamicos).

<Info>
Este ejemplo demuestra un Flow dinámico simple donde el usuario interactúa con formularios. Las transiciones entre pantallas (REGISTER → REGISTRATION_SUCCESS o REGISTRATION_DISABLED) son controladas por la respuesta de tu endpoint después del envío del formulario (`data_exchange`). Para detalles sobre cómo implementar el servidor y controlar el flujo, consulta la documentación de Flujos dinámicos.
</Info>
