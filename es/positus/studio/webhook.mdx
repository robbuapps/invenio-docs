# Webhook

Para configurar el webhook del workspace accede a: [https://studio.posit.us/workspace/configuracoes](https://studio.posit.us/workspace/configuracoes)

Todo evento entregado a tu webhook sigue el mismo envoltorio: un campo `event` con el nombre del evento, un objeto `workspace` que identifica el workspace de origen y un objeto con el recurso relacionado al evento (`number`, `template`, etc.). La entrega se realiza mediante HTTP `POST`.

**Objeto `workspace`**

| Campo            | Descripción                                                  |
| ---------------- | ----------------------------------------------------------- |
| id               | Identificador (UUID) del workspace                          |
| provider         | Proveedor del workspace: `{ "id", "name" }`                 |
| type_workspace   | Tipo del workspace: `{ "id", "name" }` (Receptivo / Prospect) |
| name             | Nombre del workspace                                        |
| business_id      | ID del Business Manager en Meta                            |
| waba_id          | ID de la WABA en Meta                                      |

## Clasificación de calidad del número

Recibe en tu webhook la calidad de cada número de WhatsApp con el fin de mantener una mejor integración con tu CRM, mejorando así la calidad de la gestión de tus números.

* Indefinido: aún no calificado
* Verde: calidad alta
* Amarillo: calidad media
* Rojo: calidad baja

### Eventos de número

| Tipo de Evento              | Descripción                                                                                                                                          |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| phone_number_status_update  | Se envía en tiempo real cada vez que se modifica el estado, la clasificación de calidad (`quality_rating`) o el límite de mensajes (`messaging_limit`) del número. |
| phone_number_quality_update | Se envía durante la sincronización periódica de los números. Utiliza un payload en formato plano (ver el ejemplo a continuación).                    |
| phone_number_sync           | Se envía durante la sincronización periódica, reflejando el estado actual del número.                                                               |

### Estado del número

| **Estado del número** | Descripción                                                  |
| --------------------- | ----------------------------------------------------------- |
| 1                     | Número esperando aprobación                                  |
| 2                     | Número esperando creación del servidor                      |
| 3                     | Número esperando autenticación del servidor                 |
| 4                     | Número esperando instalación del servidor                   |
| 5                     | Número esperando configuración del servidor                 |
| 6                     | Número esperando activación                                  |
| 7                     | Número activo                                                |
| 8                     | Número inactivo                                              |
| 9                     | Número eliminado                                             |
| 10                    | Esperando migración a Cloud Api                             |
| 11                    | Número esperando reinicio del servidor                      |
| 12                    | Número esperando finalización de la activación por el usuario |
| 13                    | Número baneado                                               |
| 14                    | Número esperando restauración                               |
| 15                    | Número transferido                                          |

### Estado de calidad (`quality_status`)

| ID | Código    |
| -- | --------- |
| 0  | UNKNOWN   |
| 1  | CONNECTED |
| 2  | FLAGGED   |
| 3  | MIGRATED  |

### Clasificación de calidad (`quality_rating`)

| ID | Descripción de la clasificación de calidad |
| -- | ------------------------------------------ |
| 0  | UNKNOWN                                    |
| 1  | GREEN                                     |
| 2  | YELLOW                                   |
| 3  | RED                                      |

## Mensaje de plantilla

Recibe notificaciones de aprobación de tus plantillas de notificaciones a través del webhook. El evento `message_template_status_update` se envía cada vez que se modifica el estado de aprobación de una plantilla.

### Estado de la plantilla

| ID | Código (code)    | Descripción       |
| -- | ---------------- | ----------------- |
| 0  | UNDEFINED        | Indefinido        |
| 1  | PENDING          | Pendiente         |
| 2  | APPROVED         | Aprobado          |
| 3  | REJECTED         | Rechazado         |
| 4  | IN_APPEAL        | En apelación      |
| 5  | PENDING_DELETION | Eliminación pendiente |
| 6  | DELETED          | Eliminado         |
| 7  | DISABLED         | Deshabilitado     |
| 8  | FLAGGED          | Señalado          |
| 9  | REINSTATED       | Restablecido      |
| 10 | PAUSED           | Pausado           |
| 11 | PAUSED           | Pausado           |
