# Webhooks de Llamadas

Los eventos de llamadas VoIP llegan en el webhook de llamadas (`calls_webhook`) de tu número, **separado del webhook de mensajes**. Cada notificación se envía con el encabezado `User-Agent: Positus`.

## Tipos de eventos

| Evento | ¿Genera webhook? | Descripción |
| --- | --- | --- |
| connect | Sí | Llamada iniciada por un usuario. Contiene identificadores de llamada, dirección y datos de sesión (SDP) para WebRTC. |
| pre_accept | No | Acción interna de Positus. No genera webhook. |
| accept | No | Acción de tu aplicación a través de API. No genera webhook. |
| reject | No | Acción de tu aplicación a través de API para rechazar. No genera webhook. |
| terminate | Sí | Llamada finalizada (por el usuario, tu aplicación o timeout). Incluye timestamps, duración y estado final. |

<Info>
Solo los eventos **connect** y **terminate** generan webhooks. Las acciones de aceptar, rechazar o terminar una llamada se realizan a través de la [API de Llamadas](/es/positus/calling/api) y no generan notificaciones en webhook.
</Info>

## Procesamiento

<Info>
El payload de webhook sigue el estándar de notificaciones de Meta. Para procesar los eventos, lee el objeto de llamada en `entry[0].changes[0].value.calls[]`. Cada llamada contiene un identificador único (`id`) que puedes usar para rastrear la llamada en tu aplicación.
</Info>
