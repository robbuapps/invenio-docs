# Flows Webhooks

To track flow updates, it is recommended that you configure a webhook in the workspace to receive real-time updates about the flows. You must [configure a webhook in the workspace](https://studio.posit.us/workspace/configuracoes) to receive these events.

## Flow Events

The table below lists all flow events that can be received through workspace webhooks:

| Event | Description |
| --- | --- |
| flow_created | This event is received when a new flow is created |
| flow_updated | This event is received when a flow is updated |
| flow_published | This event is received when a flow is published |
| flow_throttled | This event is received when a flow is throttled |
| flow_deprecated | This event is received when a flow is deprecated |
| flow_blocked | This event is received when a flow is blocked |
| flow_deleted | This event is received when a flow is deleted |
| flow_sync | This event is received when the Positus system forces a sync with Meta |

<Info>
It is important that your Webhook returns an HTTPS 200 OK response to notifications. Otherwise, the WhatsApp Business API client will consider that notification a failure and will retry after a delay.
</Info>
