## How do I invite a new user to my Workspace?

Only users registered on the platform can join your workspace; every user must first create their own personal account using a valid email. \
\
📌 Go to your workspace using the left sidebar menu and click the "Invite" button. Next, enter one or more valid email addresses separated by semicolons (`;`).

The invited user will receive an email as shown below and must click the "Accept Invitation" button. If they do not have an account yet, the system will route them to the registration screen. Once the account is created, the user will be automatically linked to your workspace.

## How do I generate an API Token?

Navigate to the "API Tokens" menu directly or access [https://studio.posit.us/minha-conta/api-tokens](https://studio.posit.us/minha-conta/api-tokens), then click on the "New Token" button.

Make sure to copy your new token to a secure location immediately. For security reasons, it will never be displayed again in the future.

![Generating a new API access token in Positus Studio](/images/positus/studio/gerar-token-api.gif)

<Info>
  **Warning!** Do not share personal access tokens with anyone! A token grants
  administrative access to all data and resources within your account.
</Info>

### How do I delete an active token?

Navigate to the "API Tokens" menu at [https://studio.posit.us/minha-conta/api-tokens](https://studio.posit.us/minha-conta/api-tokens), click on the ellipsis button (`...`) next to the target token, and click "Revoke".

![Revoking an active API token](/images/positus/studio/revogar-token-api.gif)

<Info>
  Once an API access token is revoked, it is permanently destroyed and cannot be
  recovered.
</Info>

## Creating Message Templates (HSM)

Before setting up templates, remember that Highly Structured Messages (HSMs) exist specifically to pre-format message templates that strictly comply with WhatsApp's policies, which prioritize the end-user experience. Therefore, broadcasting generic promotional or unsolicited content is forbidden. Ensure your copy does not contain language flagged as SPAM (e.g., hard promotional text or direct sales copy without authorization).

Keep these critical HSM features in mind:

1. **Rigid Structure:** They are highly structured and must comply with strict layout rules.
2. **Pre-Approval:** Every template must be reviewed and approved by WhatsApp before deployment.
3. **Opt-In Required:** Recipients must explicitly opt-in to receive these notifications beforehand.
4. **Dynamic Variables:** You can customize parameters using dynamic variables (e.g., standard customer names).

#### Structural parameters to follow:

- **Category:** Choose the use case category that best matches your message content.
- **Template Name:** This identifier can only contain lowercase alphanumeric characters and underscores (`_`). Spaces, uppercase letters, or special characters are invalid.
- **Correct Variable Formatting:** Variables must follow a sequential string pattern wrapped as `{{#}}`, where the number represents the index. Note that indices must strictly start at `{{1}}`.
- **Review Content Carefully:** Meta's approval team reviews templates standalone without knowing your broader business workflows. Proofread your copy multiple times to ensure the context is explicitly clear.

### Message Template Status Tracking

- <img

  />
  **Pending** The template is currently awaiting verification from Meta's review

- <img

    alt="Rejected status icon"
    height="16"
  />
  **Rejected** The primary reason for template rejection is using aggressive
  marketing terms or pushy sales pitches that users did not clear. Certain
  buzzwords trigger automated rejections as SPAM—avoid expressions such as
  "clearance", "limited edition", "special opportunity", or "discount". Choose
  your phrasing wisely.

![Creating an HSM message template in Positus Studio](/images/positus/studio/criar-template.gif)

<Info>
  You can enrich your templates with Emojis, as well as markdown formatting like
  bold, italics, or strikethroughs. Rich media components are also fully
  supported (Images, Documents, and Videos).
</Info>

Once an outbound HSM template successfully initiates a conversation and the user replies, a standard session window opens. You can then communicate freely using open text, rich attachments, or inline media without requiring additional templates.

### Handling Rejected Message Templates (HSM)

If a template is rejected due to SPAM policies or formatting violations, **it cannot be edited or deleted**. You will need to create and submit a brand new template. There is currently a system limit of 250 templates per WhatsApp Business Account (WABA).

## Support - Opening Tickets

You do not need to be an active paying customer to reach our helpdesk; technical support is available to all users with registered platform accounts.

To open a support request, navigate to the left sidebar menu under **Support >> New Ticket**. Select your issue category, define a clear subject title, write out your detailed message, and click "Create Ticket". Whenever possible, upload screenshots or log payloads as attachments to serve as evidence.

_If you are an agency partner managing multiple entities, make sure to select the correct target workspace from your dropdown menu before submitting._

You can also monitor closed tickets at any time by visiting the "Completed" dashboard archive.

![Opening a new technical support ticket in Positus Studio](/images/positus/studio/abrir-ticket.gif)
